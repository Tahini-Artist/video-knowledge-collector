# 视频知识收集器

## 概述

帮助创业者从视频教程中提取价值，构建个人知识库。支持小红书、抖音、B站链接解析，**完整提取所有知识点**、**结合用户身份生成个性化行动建议**、分析主题关联。

## ⚠️ 首次使用必读

**在第一次使用前，必须收集用户的关键信息：**

### 必填信息
1. **身份**：你的职业/角色是什么？
   - 示例：独立开发者、自媒体博主、产品经理、创业者、学生...

2. **业务情况**：你当前在做什么业务/项目？
   - 示例：正在做一个 AI 工具、运营一个小红书账号、开发一个 SaaS 产品...

3. **目标**：你希望达成什么目标？
   - 示例：3 个月做到 1 万粉丝、月收入突破 5 万、产品上线获得 1000 用户...

### 信息收集流程
```
首次使用时：
1. 检查是否已收集用户信息（查看 references/user-profile.md）
2. 如未收集，询问用户：
   「为了给你生成更精准的行动建议，我需要了解一下你：
   1. 你的身份是？
   2. 你当前在做什么业务/项目？
   3. 你的目标是什么？」
3. 保存到 references/user-profile.md
4. 后续使用时直接读取，无需重复询问
```

### 用户信息保存格式
```markdown
# 用户画像

## 身份
[用户身份]

## 业务情况
[用户当前业务描述]

## 目标
[用户目标]

## 更新记录
- 2026-05-01：首次收集
```

## 核心能力

### 1. 链接解析与内容提取

当用户提供视频链接时，按以下流程处理：

**支持的平台：**
- 小红书：`xiaohongshu.com/discovery/item/...`
- 抖音：`douyin.com/video/...`
- B站：`bilibili.com/video/...`

**处理流程：**
1. 解析链接，识别平台
2. 获取视频标题、博主信息
3. 尝试提取视频内容（文字稿/字幕/评论）
4. 如无法直接提取，请求用户提供截图或文字描述

**内容获取策略：**
```
优先级：
1. 直接解析网页内容（B站通常有字幕）
2. 通过搜索引擎查找视频文字版
3. 请求用户提供关键截图
4. 请求用户描述核心内容
```

### 2. 知识点提取（完整提取）

**核心原则：提取视频中的【全部】知识点，不做精简，不遗漏任何有价值的内容。**

分析视频内容后，按以下结构提取知识点：

```markdown
## 💡 完整知识点

### 一、[视频核心主题 1]

#### 1.1 [知识点标题]
- 核心概念：...
- 具体方法：...
- 实际案例：...
- 适用场景：...
- 时间点：[00:00]

#### 1.2 [知识点标题]
...

### 二、[视频核心主题 2]

#### 2.1 [知识点标题]
...
```

**提取原则：**
1. **完整性优先**：不限制知识点数量，有多少提取多少
2. **保留原始信息**：具体数字、步骤、公式、工具名称、平台规则等全部保留
3. **标注时间点**：每个知识点标注视频时间点（如：[05:23]），便于回看
4. **结构化呈现**：按主题分组，形成清晰的知识体系
5. **区分知识类型**：
   - 📌 概念性知识（理解类）
   - 🔧 实操性知识（行动类）
   - ⚠️ 注意事项（避坑类）
   - 💡 案例参考（借鉴类）

**提取深度要求：**
- 视频提到的每个观点都要提取
- 视频提到的每个方法都要记录
- 视频提到的每个工具/平台都要标注
- 视频提到的每个案例都要摘要

### 3. 个性化行动建议生成

**核心原则：结合【用户身份 + 业务情况 + 目标】，生成个性化、可执行的行动建议。**

#### 3.1 分析维度

在生成行动建议前，先分析：
1. **用户身份** → 这个知识点对用户的适用性
2. **业务情况** → 这个建议是否贴合用户当前业务
3. **目标** → 这个行动是否能推进用户的目标

#### 3.2 生成逻辑

```
知识点：「小红书笔记发布时间影响曝光」

用户身份：独立创业者
业务情况：做 AI 工具，小红书账号刚起步
目标：3 个月做到 1 万粉丝

→ 行动建议：
  「针对你的 AI 工具目标用户（打工人），建议在以下时间段发布：
   - 早高峰 7:30-8:30（通勤时间）
   - 午休 12:00-13:00（刷手机时间）
   - 晚高峰 18:00-19:00（下班时间）
   
   本周行动：连续 7 天在 12:30 发布笔记，记录数据表现」
```

#### 3.3 建议分类

**📌 立即执行（今天可做）**
- 具体的小任务，30 分钟内可完成
- 必须说明：做什么 + 为什么 + 预期效果

**📅 短期计划（本周完成）**
- 需要 1-7 天的任务
- 必须说明：具体步骤 + 时间安排 + 验证方法

**🎯 长期探索（持续投入）**
- 需要持续迭代的事情
- 必须说明：最终目标 + 阶段里程碑 + 检查频率

**❌ 不适用项（说明原因）**
- 视频中提到但不适合用户的内容
- 必须说明：为什么不适用 + 替代方案（如有）

**输出格式：**
```markdown
## ✅ 个性化行动建议

> 基于你的身份【独立创业者】、业务【AI 工具】、目标【3 个月 1 万粉丝】生成

### 📌 立即执行（今天可做）

#### 行动 1：[具体行动]
- **做什么**：[具体操作步骤]
- **为什么**：[与你的目标的关系]
- **预期效果**：[预计能带来什么]
- **所需时间**：[预估时间]

### 📅 短期计划（本周完成）

#### 行动 1：[具体行动]
- **具体步骤**：
  1. ...
  2. ...
- **时间安排**：[周几/哪个时段]
- **验证方法**：[如何判断是否成功]

### 🎯 长期探索（持续投入）

#### 行动 1：[具体行动]
- **最终目标**：[要达成什么]
- **阶段里程碑**：
  - 第 1 周：...
  - 第 2 周：...
- **检查频率**：[每天/每周检查]

### ❌ 不适用项

- **[视频中的某个建议]**：[不适用于你的原因]，替代方案：[如有]
```

### 4. 主题关联分析

分析当前视频与已有素材的主题关联：

**主题分类：**
- 流量获取
- 产品开发
- 变现方法
- 内容创作
- 用户运营
- 工具效率
- 思维认知
- 其他

**关联规则：**
1. 自动识别视频主题，分配到对应分类
2. 搜索已有素材，找出同主题内容
3. 生成 `[[双链]]` 连接相关笔记

**输出格式：**
```markdown
## 🔗 相关素材

### 同主题素材
- [[视频A]] - 相同主题：流量获取
- [[视频B]] - 相同主题：流量获取

### 互补内容
- [[视频C]] - 互补视角：产品开发
```

## 工作流程

### 完整流程

```
用户发送视频链接
    ↓
1. 解析链接，识别平台
    ↓
2. 尝试获取视频内容
    ├─ 成功 → 继续
    └─ 失败 → 请求用户补充信息
    ↓
3. 提取知识点
    ↓
4. 生成行动建议
    ↓
5. 分析主题关联
    ↓
6. 生成笔记内容
    ↓
7. 写入 Obsidian
    ├─ 新笔记 → 00-收件箱/
    └─ 分类后 → 01-视频素材/{主题}/
    ↓
8. 更新行动清单（02-行动清单/）
```

### 快速流程（仅提取知识点）

当用户只需要快速分析，不需要归档时：

```
用户提供视频链接
    ↓
提取核心知识点（3-5 条）
    ↓
输出简明摘要
```

## 使用示例

### 示例 1：收集 B 站教程

**用户：**
> 帮我分析这个视频：https://www.bilibili.com/video/BV1xxx

**处理：**
1. 解析 B 站链接
2. 获取视频标题和字幕
3. 提取知识点
4. 生成行动建议
5. 询问是否归档到 Obsidian

### 示例 2：批量整理

**用户：**
> 我有 3 个视频要整理，帮我归类：
> - 链接 1
> - 链接 2
> - 链接 3

**处理：**
1. 逐个分析
2. 汇总主题
3. 批量写入对应分类

### 示例 3：查看知识网络

**用户：**
> 我收集了哪些关于"流量获取"的视频？

**处理：**
1. 扫描 Obsidian 库
2. 筛选标签/主题
3. 列出相关笔记
4. 可视化关联关系

## 配置说明

### 首次使用

**必须完成两件事：**

#### 1. 收集用户信息（必填）

```
请初始化视频知识收集器，我的信息如下：
- 身份：独立创业者
- 业务情况：正在做一个 AI 写作工具
- 目标：6 个月内做到 1 万付费用户
```

或分步回答：
```
我的身份是：独立创业者
我当前在做：AI 写作工具
我的目标是：6 个月 1 万付费用户
```

#### 2. 配置 Obsidian 库路径（可选）

```
我的 Obsidian 库路径是：/Users/xxx/Obsidian/我的知识库
```

### 自定义主题分类

用户可以根据需要添加新的主题分类：

```
添加主题分类：AI工具、自动化
```

## 注意事项

1. **视频内容获取限制**
   - 小红书、抖音需要登录才能查看完整内容
   - B 站部分视频没有字幕
   - 无法获取时，请求用户提供截图或描述

2. **知识库路径**
   - 首次使用需要用户提供 Obsidian 库路径
   - 路径保存在 `references/config.md` 中

3. **主题关联**
   - 自动分类基于关键词匹配
   - 用户可以手动调整分类
