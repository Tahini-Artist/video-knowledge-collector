# 视频知识收集器

## 概述

帮助创业者从视频教程中提取价值，构建个人知识库。支持小红书、抖音、B站链接解析，自动提取知识点、生成行动建议、分析主题关联。

## 核心能力

### 1. 链接解析与内容提取

当用户提供视频链接时，按以下流程处理：

**支持的平台：**
- 小红书：`xiaohongshu.com/discovery/item/...`
- 抖音：`douyin.com/video/...`
- B站：`bilibili.com/video/...`

**处理流程：**
1. 解析链接，识别平台
2. 获取视频标题、博主信息
3. 尝试提取视频内容（文字稿/字幕/评论）
4. 如无法直接提取，请求用户提供截图或文字描述

**内容获取策略：**
```
优先级：
1. 直接解析网页内容（B站通常有字幕）
2. 通过搜索引擎查找视频文字版
3. 请求用户提供关键截图
4. 请求用户描述核心内容
```

### 2. 知识点提取

分析视频内容后，按以下结构提取知识点：

```markdown
## 💡 核心知识点

### 主题：[视频核心主题]

#### 要点 1：[知识点标题]
- 核心概念：...
- 具体方法：...
- 实际案例：...
- 适用场景：...

#### 要点 2：...
```

**提取原则：**
- 每个视频提取 3-7 个核心知识点
- 保留具体数字、步骤、公式等关键信息
- 标注时间点（如：[05:23]）便于回看
- 区分"概念性知识"和"实操性知识"

### 3. 行动建议生成

根据视频内容，生成三类行动建议：

**类别 A：立即执行**
- 可以马上做的小任务
- 示例："修改小红书主页简介"

**类别 B：短期计划**
- 需要 1-7 天完成
- 示例："尝试 3 种不同的封面风格"

**类别 C：长期探索**
- 需要持续投入
- 示例："建立自己的内容发布节奏"

**输出格式：**
```markdown
## ✅ 行动建议

### 📌 立即执行（今天）
- [ ] 待办事项 1
- [ ] 待办事项 2

### 📅 短期计划（本周）
- [ ] 待办事项 1
- [ ] 待办事项 2

### 🎯 长期探索（持续）
- [ ] 待办事项 1
```

### 4. 主题关联分析

分析当前视频与已有素材的主题关联：

**主题分类：**
- 流量获取
- 产品开发
- 变现方法
- 内容创作
- 用户运营
- 工具效率
- 思维认知
- 其他

**关联规则：**
1. 自动识别视频主题，分配到对应分类
2. 搜索已有素材，找出同主题内容
3. 生成 `[[双链]]` 连接相关笔记

**输出格式：**
```markdown
## 🔗 相关素材

### 同主题素材
- [[视频A]] - 相同主题：流量获取
- [[视频B]] - 相同主题：流量获取

### 互补内容
- [[视频C]] - 互补视角：产品开发
```

### 5. Obsidian 自动归档

将处理结果写入 Obsidian 知识库：

**知识库结构：**
```
Obsidian/
├── 00-收件箱/           # 新收集的视频
├── 01-视频素材/         # 按主题分类
│   ├── 流量获取/
│   ├── 产品开发/

├── 03-知识网络/         # 主题关系图

```

## 工作流程

### 完整流程

```
用户发送视频链接
    ↓
1. 解析链接，识别平台
    ↓
2. 尝试获取视频内容
    ├─ 成功 → 继续
    └─ 失败 → 请求用户补充信息
    ↓
3. 提取知识点
    ↓
4. 生成行动建议
    ↓
5. 分析主题关联
    ↓
6. 生成笔记内容
    ↓
7. 写入 Obsidian
    ├─ 新笔记 → 00-收件箱/
    └─ 分类后 → 01-视频素材/{主题}/
    ↓
8. 更新行动清单（02-行动清单/）
```

### 快速流程（仅提取知识点）

当用户只需要快速分析，不需要归档时：

```
用户提供视频链接
    ↓
提取核心知识点（3-5 条）
    ↓
输出简明摘要
```

## 使用示例

### 示例 1：收集 B 站教程

**用户：**
> 帮我分析这个视频：https://www.bilibili.com/video/BV1xxx

**处理：**
1. 解析 B 站链接
2. 获取视频标题和字幕
3. 提取知识点
4. 生成行动建议
5. 询问是否归档到 Obsidian

### 示例 2：批量整理

**用户：**
> 我有 3 个视频要整理，帮我归类：
> - 链接 1
> - 链接 2
> - 链接 3

**处理：**
1. 逐个分析
2. 汇总主题
3. 批量写入对应分类

### 示例 3：查看知识网络

**用户：**
> 我收集了哪些关于"流量获取"的视频？

**处理：**
1. 扫描 Obsidian 库
2. 筛选标签/主题
3. 列出相关笔记
4. 可视化关联关系

## 配置说明

### 首次使用

1. 确认 Obsidian 库路径
2. 初始化目录结构
3. 创建模板文件

**初始化命令：**
```
请初始化视频知识收集器，我的 Obsidian 库路径是：/path/to/obsidian
```

### 自定义主题分类

用户可以根据需要添加新的主题分类：

```
添加主题分类：AI工具、自动化
```

## 注意事项

1. **视频内容获取限制**
   - 小红书、抖音需要登录才能查看完整内容
   - B 站部分视频没有字幕
   - 无法获取时，请求用户提供截图或描述

2. **知识库路径**
   - 首次使用需要用户提供 Obsidian 库路径
   - 路径保存在 `references/config.md` 中

3. **主题关联**
   - 自动分类基于关键词匹配
   - 用户可以手动调整分类
